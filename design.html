Abstract

The website I created is very soccer oriented and specifically tailored to those who are interested in refereeing soccer. The homepage provides links to all the other pages and contains an external CSS along with JavaScript that changes one of the pictures on the bottom (up to you to see how… maybe click?). I have one page devoted to the two most famous referees in the world; Mark Geiger and Pierluigi Collina. This page uses external JavaScript to deliver a very simple alert to the user and an external CSS.

My website continues with an interface that is a quiz about soccer and referees in general. The interface is the only page with an internal CSS which is the background picture. Currently it also has the JavaScript code internal. The whole quiz is based off of JavaScript and the questions are written in HTML using radio buttons. Once the user answers the question and hits the button in the bottom the function GetScore tells the user what percentage they got correct. The user can then clear the answers and attempt again.

The ProReferee page takes a look at why this organization is so effective using the vision and mission statement. This page also has a very simple external CSS and has no JavaScript at this point. In general, this web page serves as a great showing of the JavaScript, CSS, and HTML I have learned up to this point.

Design Goals

  1. To build an interface that challenges users asking general question with regard to soccer and refereeing. Needs to include HTML and JavaScript that is challenging for me to create but easy enough to complete in the time frame given.
  2. To use storage in the form of cookies to determine what type of user is on the site and what type of soccer knowledge they have. What the user enters should be shown on the homepage as well as given as an alert each time the user returns to the home page.
  3. Use JavaScript to make new functions and methods that I have never tried before. These should be included in my cookies but most should occur in my interface quiz. Once again I want to challenge my skills but have a working interface by the time the assignment is due.

Testing

While testing the website I began to see that on the interface the background image of the referee was getting in the way of the quiz. The user wouldn’t be able to actually read the questions that were being asked and this effectively ruined the purpose of my interface. I started to look for a way to move the image over and found multiple solutions. Another problem I kept running into was that I would save the changes to my html files and load the webpage but github still hadn’t made the changes. This made it incredibly hard to tell in a timely manner if my changes were making any impact to the webpage. I really just had to learn to be patient with the page.

Alpha

For example, the cookie that asks what type of person is visiting the page would work for the first time when it asked if the person what profession they have. Once the page reloaded though the cookie would complete the “onload="checkCookie()" “ function. This was a struggle to try and resolve and unfortunately I still have not discovered a way to fix this issue.
  2. I ran into the same issue with my interface and once again decided to leave the JavaScript in the file with the HTML.

Eventually I found an example that I believed would work. Unfortunately, when I put this portion of code into my webpage it would not line the pictures up and actually scatter them across the page. After about an hour of trying to fix this code I actually just scrapped it and looked for a new option. I ended up finding some examples that actually included a way to change the pictures when people clicked on them so it was nice to include that and see something new.

Beta

How The Test Was Conducted

To conduct this test, I started with some referees that would actually understand the content of the page so they could tell me the good and the bad of the page. I also included some of the students in the web and design class that could give me some helpful tips about the actual HTML, CSS, and JavaScript that I was currently using and could be using to make the page better. For each group of people, I consulted with I simply gave them the website URL and asked them to give me three things they liked and three things they didn’t like about the content and design of the webpage. The reason I didn’t give these people a summary of what the website is because I wanted it to be realistic to what people in the real world that just came across the page would be seeing. This way if I needed to include more information about the purpose of the website I would know from these tests and not after I put the fully functional website out for everyone to see.

What they found

  1. Multiple users complained about a few of the pages and not being able to read the content because of the colors on the screen—Brown background with black text just doesn’t work for some people I guess. The change I made was simple, I just took the CSS files and edited them to make the webpage a more user friendly color scheme.
  2. One user actually discovered an error in my interface. He noticed that one of my answers that I had in the quiz was not showing up as correct. At first I couldn’t figure out why this would be the case but then I realized that I had capitalized the correct answer with the radio button but not in the answer I put in the correct answers up top.
  3. The last recommendation I had was to include images of Geiger and Collina. The referees that I talked with said that giving people an image of the referees can help make them look more recognizable and just looks better on the webpage. This was easy all I had to do was throw the image into the HTML not an issue at all.
